## Linux - add / del sudo user
### Добавление пользователя

Сохрани скрипт и дай права на выполнение

```
chmod +x create_user.sh
```

Запусти с именем пользователя и путём к .pub ключу

```
sudo ./create_user.sh john /tmp/john_id_rsa.pub
```


Что делает скрипт:

1. Проверяет, что запущен от root, переданы оба аргумента, и файл ключа существует.
2. Создаёт пользователя с домашней директорией и bash-оболочкой. Если пользователь уже есть — не падает, а сообщает об этом.
3. Добавляет в группу sudo.
4. Создаёт ~/.ssh/authorized_keys и вставляет ключ — с проверкой на дубликат.
5. Выставляет правильные права: 700 на .ssh/, 600 на authorized_keys


### Удаление пользователя

Удалить пользователя, домашнюю папку оставить

```
sudo ./delete_user.sh john
```

Удалить пользователя вместе с домашней папкой

```
sudo ./delete_user.sh john --remove-home
```

Что делает скрипт сверх простого userdel:

1. Защита — нельзя случайно удалить root, daemon и других системных пользователей.
2. Проверка активных сессий — если юзер залогинен, предупреждает и спрашивает подтверждение. При согласии завершает его процессы через pkill.
3. Автобэкап — перед удалением всегда пакует /home/<user> в архив /var/backups/john_home_20260414_120000.tar.gz. Даже если потом удалишь папку — данные не потеряешь.
4. Очистка sudoers — удаляет /etc/sudoers.d/<user>, если такой файл существует.
